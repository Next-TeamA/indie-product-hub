- CTA 패턴

레퍼런스를 아래 유형으로 분류해.

### **1. 기능 출시형**

새 기능이나 업데이트를 알리는 글.

### **2. 문제 해결형**

“내가 이런 문제를 겪어서 이걸 만들었다” 유형.

### **3. 재홍보/리마인드형**

이미 만든 프로젝트를 다시 소개하는 글.

너희 서비스에 특히 중요함.

### **4. 피드백 요청형**

서비스를 보여주고 의견을 구하는 글.

### **5. 성과 공유형**

사용자 수, 방문자 수, 매출, 가입자 수 등 수치를 공유하는 글.

### **6. 개발 로그/진행상황형**

오늘 무엇을 만들었는지 공유하는 글.

이 중 MVP에는 최소 3개만 넣어도 됨.

추천 우선순위는:

1. 재홍보/리마인드형
2. 문제 해결형
3. 기능 출시형
4. 피드백 요청형
5. 개발 로그형

왜냐면 너희 핵심이 **방치된 프로젝트를 다시 홍보하는 것**이니까.
